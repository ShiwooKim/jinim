# 지님 브랜드 웹(MVP) 기획 작업 계획서

> 기준 문서: `brand-def.md`  
> 참고: `bi-design-plan.md`(로고·컬러), `project-logic.md`, `project-approach-rule.md`, `슬라이드-기획서.md` 등 동일 폴더 자료

---

## 1. 목적과 최우선 목표

**목적:** 지님의 브랜드 철학을 한 번에 이해시키는 **홍보용 단일 랜딩**을 만들고, 새로 구매한 **`jinim.kr`** 에 안정적으로 배포한다.

**최우선 성공 기준 (이 순서를 지킨다):**

1. `jinim.kr` 접속 시 로딩·모바일·PC 모두 문제 없이 본문이 보인다.
2. 첫 화면 질문과 스크롤 흐름만으로 **brand-def §3·§6·§7의 축**(문제의식 → 핵심 질문 → 대표 메시지 → 무엇을 하는 서비스인지)이 전달된다.
3. 상단에 **지님 로고**, 하단에 **최소 구성의 푸터**가 일관되게 적용된다.

범위 밖으로 밀어낼 것: 회원·로그인·앱 기능·폼으로 이어지는 복잡한 리드 수집 파이프라인 등은 **이번 MVP 기획의 필수 산출물에 넣지 않는다.** (후술: 확장만 고려)

---

## 2. 범위 / 비범위

### 2.1 이번에 만든다 (In scope)

- **단일 URL·단일 페이지(또는 짧은 앵커 스크롤)** 구조의 브랜드 랜딩.
- **필수 첫 화면 카피:**  
  **「당신에게 있어 지님의 가치란 무엇입니까?」** — `brand-def.md` §6 핵심 질문을 히어로에 반드시 포함한다.
- **스크롤 내러티브:** 위 질문에서 시작해, 아래로 넘길수록 철학이 **짧은 문장·강한 한 줄**로 쌓이도록 섹션 설계 (구구절절한 설명문 지양 — `brand-def.md` §19·§20, `project-approach-rule.md` 톤 원칙과 정합).
- **헤더:** 좌측(또는 중앙 정렬 브랜드 규칙에 맞게) **지님 로고** 배치. 네비는 최소(예: 최상단 앵커만, 또는 없음).
- **푸터:** **간소화** — 저작·연락·SNS 중 필요한 것만 1줄~소수 링크 수준 (`brand-def.md`의 과시·랭킹 문화와 어긋나는 요소 넣지 않음).
- **반응형:** **PC·모바일** 동일 콘텐츠, 레이아웃만 유연하게 (`project-logic.md` 3단계 “첫 화면/랜딩 구성”과 연결).

### 2.2 이번에 만들지 않는다 (Out of scope)

- 로그인, 회원가입, 마이페이지, 실제 **지님 카드** 입력·저장 기능.
- 서버 연동이 필요한 대기자 명단·DB 연동(원하면 **정적 링크 하나** 수준만 나중에 옵션으로 검토).

### 2.3 나중에 붙일 것만 “고려”한다

- 인증·제품 진입 연결 시 **같은 도메인 하위 경로** 또는 서브도메인 분리 여지.
- 헤더에 향후 “시작하기” 등 CTA 자리만 **시각적 여백·컴포넌트 분리**로 남겨 두기 (기능은 미구현 가능).

## 3. 기획력 집중 방향: 브랜드 철학 우선

모든 의사결정은 **`brand-def.md`** 를 1차 기준으로 한다.

| 초점 | 근거 섹션 | 이번 랜딩에서 할 일 |
|------|-----------|---------------------|
| 왜 지금인가 | §3 문제의식 | 1~2개 섹션: AI·생성 시대의 불안 → **실물·시간**으로의 시선 (길게 쓰지 말고 한 덩어리 메시지) |
| 무엇인가 | §2 한 문장, §4 미션 | 서비스 정의·미션을 **한 화면 분량**으로만 |
| 무엇이 다르가 | §10 포지셔닝 | 기존 vs 지님 표는 **모바일에서 읽히게** 줄 수·타이포 조정 (과도한 표 대신 3쌍만 써도 됨) |
| 어떤 말투인가 | §19 톤앤매너 | 담백·정제·조용하지만 선명하게; 과한 감성·과시 배제 |
| 사용자에게 남기는 한 줄 | §7 대표 메시지 | **「내가 살아온 시간은 여전히 내 것이다.」** 및 보조 메시지 1~2개만 선별 배치 |

**하지 말아야 할 것**은 `brand-def.md` §22를 체크리스트로 두고, 랜딩 카피·이미지 모두에서 걸러낸다 (가격·시세 중심, 사치 과시 톤 등).

## 4. 정보 구조(IA): 한 페이지 스크롤 제안

아래는 **스토리 순서** 예시. 카피는 기획 단계에서 문장 수를 제한하고, 구현 전 확정한다.

1. **Hero**  
   - 필수: **「당신에게 있어 지님의 가치란 무엇입니까?」**  
   - 보조 한 줄: 서비스 성격을 오해 없이 (예: 한 문장 정의 — `brand-def.md` §2).  
   - 시각: 여백 + 브랜드 톤 이미지 1장 (§7 참고 이미지 방향).

2. **문제의식 (짧게)**  
   - 생성·복제 시대의 질문 → 지님이 잡는 답의 방향 (**실제로 지니고 살아온 물건**) — `brand-def.md` §3.

3. **지님의 약속**  
   - 미션 한 문장 §4, 최종 기준 문장 중 1개 선택 §24.

4. **가치 / 관점**  
   - §8 브랜드 가치 5개는 **아이콘·짧은 레이블**로만, 또는 그중 3개만 대표 노출.

5. **우리가 묻는 것**  
   - §6 보조 질문들 중 **2~3개만** 인용해 “사람에 대한 질문”임을 보여 줌.

6. **다음에 올 것 (선택·아주 짧게)**  
   - 제품은 **나의 지님 카드** 중심이라는 사실만 (`brand-def.md` §13). 기능 나열은 최소화.

7. **마무리 메시지**  
   - 대표 메시지 반복 또는 영문 한 줄 (`brand-def.md` §7, 영문 후보 중 1).

8. **Footer**  
   - 로고 반복 없이 텍스트 링크 최소화 (저작권, 필요 시 메일 또는 고정 SNS).

앵커 ID를 두면 “첫 질문 → 스크롤” 흐름이 명확해진다.

## 5. UI 구조 규칙 (요청사항 반영)

| 영역 | 규칙 |
|------|------|
| 상단 바 | **지님 로고** 노출. 높이·여백은 모바일 터치 영역 고려. |
| 푸터 | **간소화** — 컬럼 여러 개·사이트맵 형태는 피함. |
| CTA | MVP에서 과도한 버튼보다 **한 개의 차분한 CTA**(예: “소식 받기” 정적 링크는 선택) 또는 CTA 없이 브랜드만 전달도 가능. |
| 로그인 | **없음.** 향후 자리만 헤더 컴포넌트 분리로 대비. |

시각 규칙은 `bi-design-plan.md` 와 정렬: **Archive Brown `#7A624C`**, 배경 **Warm Ivory `#F5F1EA`**, 본문 **Charcoal `#2B2926`** 등. 로고 파일 형식(SVG/PNG)과 최소 사이즈는 BI 문서에 따름.

## 6. 카피 작업 원칙 (짧고 강하게)

- **긴 단락 금지:** 섹션당 제목 1 + 본문 2~3문장 상한을 기획 시 스스로 제한한다.  
- **한 섹션 = 한 메시지.** `brand-def.md` §20의 “브랜드 철학용”과 “제품 경험용”을 혼동하지 말고, 이번 랜딩은 전자 위주·후자는 다음 단계용으로 분리.  
- 반복 문구는 **대표 메시지·핵심 질문**에만 허용한다.

검수 질문 예시:

- 이 문장을 지운 뒤에도 이 섹션이 의미를 잃는가? → 아니면 삭제 후보.
- 가격·브랜드명 없이도 이 섹션이 성립하는가? (`brand-def.md` §10)

## 7. 이미지 전략 (브랜드에 맞는 비주얼 적극 활용)

**방향:** 사진 스톡만으로 기복하지 말고, **브랜드 톤에 맞는 이미지를 생성·편집해 섹션별로 배치**한다. 모티프는 `brand-def.md` §3·§5·§9와 `bi-design-plan.md`(물성, 시간, 소품함, 조용한 고급스러움)와 맞출 것.

| 용도 | 제안 |
|------|------|
| Hero | 따뜻한 아이보리 톤 배경 위 질문 타이포 + 은은한 오브제(손·시계·가죽 질감 등 **과시 없이**) |
| 중간 섹션 | “시간·기록·소품” 은유 이미지 소량 반복 사용 금지 → **섹션마다 1 컨셉** |
| 일관성 | 컬러 그레이딩을 팔레트에 맞추거나 모노톤+브라운 포인트 |

저작권: 생성 이미지 사용 시 도구·라이선스 정책 준수. 실사 플랜이면 직접 촬영 소품만 사용 (`brand-def.md` §11 타깃 정서와 맞는 물건).

## 8. 반응형·품질 기준

- **모바일 우선**으로 레이아웃 설계 후 PC에서 여백 확장.  
- 폰트 크기·줄간격으로 가독성 확보; 표가 있으면 모바일에서는 카드 스택으로 바꿀지 기획 단계에서 결정.  
- 이미지는 `loading`·적절한 해상도·CLS 최소화를 구현 단계 체크리스트에 넣는다.

---

## 9. 배포·도메인 (`jinim.kr` 최우선)

기획서 단계에서 확정해 둘 항목:

- 호스팅 방식(정적 호스팅 여부), HTTPS, SSL.
- **메타 타이틀·디스크립션·OG 이미지** 한 벌 (`project-logic.md` “기본 SEO/공유 이미지”와 연계). 문구는 `brand-def.md` §2·§7에서 발췌.
- 배포 후 실제 단말 2종 이상에서 스크롤·터치 확인.

## 10. 작업 단계와 산출물

| 단계 | 내용 | 산출물 |
|------|------|--------|
| A | IA·섹션 순서·앵커 이름 확정 | 랜딩 와이어 목차 1페이지 |
| B | 섹션별 카피 초안 (문장 수 제한) | 카피 시트 (히어로 필수 문구 포함) |
| C | BI 적용 규칙 확정 | 컬러·타이포·로고 사용 메모 (`bi-design-plan.md` 인용) |
| D | 이미지 리스트·프롬프트/촬영 리스트 | 이미지 스펙 표 |
| E | 디자인 (Figma 등) 또는 바로 구현 | 시안 또는 컴포넌트 단위 화면 |
| F | 반응형 구현·이미지 삽입·미세 카피 조정 | 코드/빌드 산출물 |
| G | `jinim.kr` 연결·SEO·공유 테스트 | 배포 체크리스트 완료 기록 |

**의존 순서:** B(카피)와 D(이미지)는 C 톤과 동시에 맞물리므로, **히어로·마무리 메시지부터 확정**한다.

## 11. 리스크와 완화

| 리스크 | 완화 |
|--------|------|
| 카피가 길어져 철학이 흐림 | §6 원칙 + 섹션당 문장 상한 |
| 이미지가 고가·과시로 읽힘 | 소품·사용 흔적·차분한 조명; `brand-def.md` §22 체크 |
| 범위 확대(회원·폼·대시보드) | 본 문서 §2 비범위를 문서 상단에 고정하고 이슈 트래킹 시 반려 기준으로 사용 |

## 12. 이번 MVP에서의 “완료” 정의

다음을 만족하면 브랜드 웹 MVP 기획·실행 1차는 완료로 본다.

1. `jinim.kr` 에 랜딩이 공개된다.  
2. 첫 화면에 **「당신에게 있어 지님의 가치란 무엇입니까?」** 가 명확히 보인다.  
3. 스크롤만으로 `brand-def.md` 의 핵심 축이 **짧은 문장**으로 이해된다.  
4. 로고·간소 푸터·반응형·브랜드 톤 이미지가 적용된다.  
5. 로그인 없이 목표가 흔들리지 않았다.

---

*본 작업 계획서는 첫 배포와 브랜드 설득을 최우선으로 하며, 제품 MVP(`brand-def.md` §13)는 후속 단계에서 별도 일정으로 연결한다.*
