## 1. 기획 요약

### 1.1 한 줄 결론

**첫 화면 질문으로 들어와, 스크롤만으로 지님이 무엇을 믿는지 짧게 이해하고 나간다.** 로그인·제품 기능 없이 **브랜드 설득**에 자원을 쓴다.

### 1.2 최우선 목표

1. `jinim.kr` 에 **안정 배포** (HTTPS, 모바일·PC 정상 표시).
2. 히어로에 필수 카피 **「당신에게 있어 지님의 가치란 무엇입니까?」** 노출.
3. 상단 **로고** · 하단 **간소 푸터** · **반응형** · 브랜드 톤 **이미지** 적용.

### 1.3 범위

| 포함 | 제외 |
|------|------|
| 단일 URL 스크롤 랜딩 | 로그인·회원·지님 카드 입력·DB 연동 |
| 정적 콘텐츠 + 선택적 외부 링크 1개(예: 문의 메일) | 대기자 폼·복잡 리드 파이프라인(필요 시 후속) |

향후 제품 연결을 위해 헤더는 **컴포넌트 분리**, CTA 자리만 여유 있게 설계 가능.

---

## 2. 타깃과 전달 목표

### 2.1 타깃 (`brand-def.md` §11 정합)

- 일상 소품에 애착이 있거나, **물건의 의미**를 한 번이라도 생각해 본 사람.
- 가격·과시 피로가 있고, **나다운 선택·시간**에 공감하는 사람.

### 2.2 이 페이지에서의 성공

| 사용자 행동 | 기대 결과 |
|-------------|-----------|
| 첫 3초 | 질문과 브랜드 톤을 인지한다. |
| 스크롤 | “무엇이 진짜 내 것인가”에 대한 지님의 답(실물·시간·기록)을 이해한다. |
| 마지막 | 대표 메시지 **「내가 살아온 시간은 여전히 내 것이다.」** 를 기억하거나 스크린샷할 만큼 남는다. |

---

## 3. 사용자 여정 (스크롤 내러티브)

```
진입 → 핵심 질문(히어로) → 시대 맥락(짧게) → 지님이란 무엇인가 → 우리가 보는 가치 → 우리가 묻는 것 → (선택) 곧 있을 경험 한 줄 → 마무리 메시지 → 푸터
```

구구절절한 설명보다 **섹션당 메시지 1개** 원칙 (`../archive/planning-early/브랜드웹-MVP-작업계획서.md` §6).

## 5. 화면별 기획 (레이아웃 · 카피 초안)

카피는 `brand-def.md` 에서 발췌·압축한 **확정 후보**다. 디자인 단계에서 한두 단어 조정 가능하나, **히어로 질문**은 변경하지 않는다.

### 5.1 Header (전 구간 고정 또는 스크롤 후 축소)

- **좌측:** 지님 로고 (워드마크 또는 BI 확정안, `bi-design-plan.md`).
- **우측:** 비워 두거나, 필요 시 향후 “시작하기” 자리만 여백. **로그인 버튼 없음.**
- 배경: 반투명 또는 Warm Ivory와 동일 톤으로 자연스럽게 이어지게 처리.

### 5.2 `#hero` 히어로

| 요소 | 내용 |
|------|------|
| 메인 카피 (필수) | **당신에게 있어 지님의 가치란 무엇입니까?** |
| 서브 카피 | AI가 무엇이든 만들어내는 시대에, 사람이 실제로 지니고 살아온 물건의 의미와 시간을 기록하는 취향 아카이브, 지님. (`brand-def.md` §2) |
| 시각 | 전면 여백 + 브랜드 톤 히어로 이미지 1점 (§9 참조). 모바일에서는 이미지 하단 또는 배경 약하게. |

### 5.3 `#why` 지금, 우리가 묻는 것

**목적:** 문제의식을 길게 설명하지 않고 한 덩어리로 전달 (`brand-def.md` §3).

| 역할 | 카피 초안 |
|------|-----------|
| 제목 | 무엇이 진짜 내 것인가 |
| 본문 (2~3문장 이내) | 많은 것이 생성되고 복제되는 시대에, 우리는 다시 묻게 된다. 진짜 내 것은 무엇인가. 지님은 그 질문에 **실제로 지니고 살아온 물건**이라는 방향으로 답을 찾는다. 오래 찬 시계, 매일 쓰는 지갑처럼—시장 가격과 다른, **개인의 시간**이 그 안에 있다. |

### 5.4 `#what` 지님이 하는 일

| 역할 | 카피 초안 |
|------|-----------|
| 제목 | 지님이 하는 일 |
| 정의 | 내가 오래 지닌 물건을 통해 나의 시간, 취향, 정체성을 기록하는 서비스. (`brand-def.md` §2 제품 표현) |
| 미션 | 사람이 오래 지닌 물건에 담긴 시간과 의미를 기록하고, 각자가 자기 삶의 가치를 다시 발견하도록 돕는다. (`brand-def.md` §4) |
| 보조 한 줄 | 더 많이 사게 하는 서비스가 아니라, 이미 지니거나 오래 지니고 싶은 물건의 **의미를 발견**하게 하는 서비스. |

### 5.5 `#difference` 무엇이 다른가

표 전체 대신 **3쌍**만 노출해 모바일 가독성을 확보한다. (`brand-def.md` §10)

| 기존의 시선 | 지님의 시선 |
|-------------|-------------|
| 이 물건은 얼마인가 | 나에게 어떤 **의미**인가 |
| 어떤 브랜드인가 | **왜** 이 물건을 선택했는가 |
| 새로 샀는가 | 앞으로도 **계속 지니고 싶은가** |

마무리 한 줄: 지님은 소비를 부추기는 서비스가 아니라, **소유의 의미를 다시 묻는** 서비스다.

### 5.6 `#values` 바라보는 가치

5개 중 **전부 아이콘+한 줄** 또는 **대표 3개** 택일.

| 라벨 | 한 줄 (`brand-def.md` §8) |
|------|---------------------------|
| 소유보다 의미 | 왜 지니는가 |
| 자랑보다 이야기 | 개인의 시간을 기록 |
| 구매보다 발견 | 나다운 물건을 발견 |
| 컬렉션보다 태도 | 선택과 삶의 방식 |
| 생성보다 지녀온 시간 | AI가 대체하지 못하는 경험 |

### 5.7 `#questions` 더 묻는 질문들

**목적:** 핵심 질문이 “물건이 아니라 사람”에 대한 것임을 보여 준다 (`brand-def.md` §6).

- 왜 이 물건을 선택했는가?
- 왜 비싸지 않아도 소중한가?
- 앞으로도 계속 지니고 싶은 이유는 무엇인가?

(리스트 3개, 따옴표 스타일 또는 짧은 줄글.)

### 5.8 `#next` 다음에 *(선택 섹션 — 없어도 완결 가능)*

| 역할 | 카피 초안 |
|------|-----------|
| 제목 | 곧 만나는 경험 |
| 본문 한 줄 | 처음부터 커뮤니티가 아니라, **나의 지님 카드**로 내 물건을 기록하고 나만의 아카이브를 만든다. (`brand-def.md` §13 요약) |

기능 목록(등록·질문·카드 등)은 나열하지 않거나 최대 1문장으로만.

### 5.9 `#closing` 남기는 말

| 역할 | 카피 초안 |
|------|-----------|
| 대표 메시지 | **내가 살아온 시간은 여전히 내 것이다.** (`brand-def.md` §7) |
| 보조 | 생성된 것보다 지녀온 것. |
| 영문 한 줄 | Find my Time. *(또는 Things that hold your time. — §7 후보 중 택1)* |

선택 CTA: 없음, 또는 차분한 텍스트 링크 “문의” → `mailto:` (주소 확정 시 반영).

### 5.10 Footer

| 포함 | 비포함 |
|------|--------|
| © 연도 지님(Jinim) 또는 법적 표기 | 로고 반복, 다단 사이트맵 |
| 필요 시: 메일, 고정 SNS 1개 | 가격·랭킹·과시 문화와 어긋나는 카피 |

## 6. 톤앤매너 (`brand-def.md` §19 반영)

- **지향:** 담백, 정제, 따뜻하지만 조용하고 선명함.
- **지양:** 과시, 소비 유도, 과한 감성, 브랜드·가격 중심 문장.

랜딩 전체 검수 시 `brand-def.md` §22 “하지 말 것” 체크.

## 8. 이미지 기획 (섹션별)

| 섹션 | 컨셉 | 금지 |
|------|------|------|
| Hero | 아이보리 톤, 질문이 전면. 손·가죽·시계 등 **일상 소품의 질감**, 과시 없음 | 명품 로고 클로즈업, 조명 과한 럭셔리 스틸 |
| Why / What | 시간·기록 은유(그림자, 책갈피, 소품함 등) 한 컨셉 | 스톡 반복으로 톤 들쭉날쭉 |
| Values | 추상적 패턴 또는 단색 구역으로 텍스트 우선 | 이미지가 라벨보다 앞서는 구도 |
| Closing | 여백 많은 정적 장면 | 텍스트 가림 |

생성 AI 사용 시 라이선스 준수. 섹션당 **대표 이미지 1컨셉** 원칙 (`../archive/planning-early/브랜드웹-MVP-작업계획서.md` §7).

## 9. 반응형 · 품질

- **모바일 우선:** 단열 → 태블릿·PC에서 최대 폭 제한(예: 640~720px 본문 폭) + 여백.
- `#difference` 표: 좁은 화면에서는 카드 스택(행 3개 → 세로 3카드).
- 이미지: 대응형 `srcset`, 적절한 압축, CLS 방지(비율 영역 확보).
