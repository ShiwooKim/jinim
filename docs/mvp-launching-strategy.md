아래는 **지님 MVP 서비스 인입 전략 기획서** 초안이야.
핵심은 “많은 사람을 데려오는 전략”이 아니라, **작은 진심을 가진 사람들의 첫 지님 카드를 모으는 전략**으로 잡았어.

제품 정책·게스트 한도·로그인·고객향 용어는 **`docs/mvp-spec-def.md`**를 우선한다. 본 문서와 충돌하면 스펙 문서 기준으로 맞춘다.

````markdown
# 지님 MVP 서비스 인입 전략 기획서

## 1. 전략의 전제

지님은 모두를 위한 서비스가 아니다.

지님은 물건을 많이 가진 사람을 위한 서비스도 아니고, 비싼 물건을 자랑하기 위한 서비스도 아니다.  
지님은 오래 지닌 물건 하나에 담긴 이유를 말할 수 있는 사람을 위한 서비스다.

누군가에게는 지나치게 진지해 보일 수 있다.  
누군가에게는 “굳이 이런 걸 왜 기록하지?” 싶은 서비스일 수 있다.

하지만 어떤 사람에게는 다르다.

오래 찬 시계 하나, 버리지 못한 지갑 하나, 매일 쓰는 펜 하나, 낡았지만 손이 가는 가방 하나를 떠올리며  
“이건 그냥 물건이 아니라 내 시간이 담긴 물건이다”라고 느끼는 사람.

지님의 첫 사용자는 바로 이런 사람들이다.

> 지님은 모두를 위한 서비스가 아닙니다.  
> 오래 지닌 물건 하나에 담긴 이야기를 가진 사람을 위한 서비스입니다.

---

## 2. MVP 인입 전략의 핵심 목표

MVP 단계에서 지님의 목표는 대규모 가입자를 모으는 것이 아니다.

초기 목표는 명확하다.

> 좋은 지님 카드 30~50개를 모으는 것.

이때 “좋은 지님 카드”란 비싼 물건을 등록한 카드가 아니다.  
그 물건을 왜 지니고 있는지, 그 사람의 취향과 시간이 조금이라도 드러나는 카드다.

### MVP 초기 목표

| 목표 | 설명 |
|---|---|
| 첫 지님 카드 수집 | 초기 사용자들의 실제 물건 이야기 확보 |
| 카드 생성 경험 검증 | 질문에 답하고 카드를 만드는 과정이 자연스러운지 확인 |
| 공유·취향 읽기 검증 | 취향 읽기 결과와 공유 URL에 반응하는지, 소품함·추가 기록을 위해 가입으로 이어지는지 확인 |
| 타깃 반응 검증 | 어떤 사람들이 가장 진심 있게 반응하는지 확인 |
| 브랜드 톤 검증 | 과시가 아닌 이야기 중심 커뮤니티가 가능한지 확인 |

---

## 3. 핵심 타깃 정의

지님의 초기 타깃은 “물건을 좋아하는 사람”보다 더 좁게 정의해야 한다.

초기 타깃은 다음과 같다.

> 물건을 통해 자기 취향과 시간을 설명할 수 있지만,  
> 인스타식 자랑이나 가격 중심 커뮤니티에는 피로감을 느끼는 사람.

---

## 4. 초기 타깃의 특징

### 4.1 이런 사람들이 쓴다

| 특징 | 설명 |
|---|---|
| 오래 지닌 물건이 있다 | 시계, 지갑, 펜, 가방, 안경, 카메라, 노트 등 쉽게 버리지 못하는 물건이 있음 |
| 물건에 의미를 부여한다 | 가격보다 “왜 이걸 계속 쓰는지”를 중요하게 생각함 |
| 취향을 설명하고 싶어 한다 | 단순 자랑보다 자기 선택의 이유를 말하고 싶어 함 |
| 과시형 SNS에 피로하다 | 인스타식 소비 인증, 고가 브랜드 자랑, 좋아요 경쟁을 불편하게 느낌 |
| 조용한 공유를 선호한다 | 시끄럽게 드러내기보다, 취향이 맞는 사람에게 보여주고 싶어 함 |
| 기록에 호감이 있다 | 노션, 일기, 사진첩, 컬렉션 정리, 라이프로그 등에 익숙하거나 관심 있음 |
| 자기 물건을 다시 보고 싶어 한다 | 이미 가진 물건의 의미를 정리하는 행위 자체에 만족을 느낌 |

---

## 5. 인스타 자랑을 싫어하는 사람의 심리

지님의 중요한 초기 타깃은 “보여주고 싶지만 자랑처럼 보이긴 싫은 사람”이다.

이 사람들은 자기 취향을 드러내고 싶은 마음이 있다.  
하지만 일반적인 SNS 방식은 불편하다.

### 기존 SNS에서 느끼는 불편함

| 불편함 | 설명 |
|---|---|
| 자랑처럼 보일까 봐 부담스럽다 | 좋은 물건을 올리면 과시처럼 보일까 걱정함 |
| 가격 중심으로 읽힌다 | 내가 왜 좋아하는지보다 얼마짜리인지가 먼저 보임 |
| 좋아요 경쟁이 싫다 | 물건의 의미보다 반응 수가 중요해지는 구조가 피로함 |
| 맥락을 담기 어렵다 | 사진 한 장으로는 그 물건을 오래 지닌 이유를 설명하기 어려움 |
| 고가 브랜드 중심이 된다 | 비싸고 유명한 물건이 더 주목받는 분위기가 불편함 |

지님은 이 불편함에 대한 대안이 될 수 있다.

> 지님은 물건을 자랑하는 공간이 아니라,  
> 물건에 담긴 이유를 기록하는 공간이다.

---

## 6. 초기 커뮤니티의 방향: 작은 진심을 가진 사람들

지님은 처음부터 큰 커뮤니티를 만들려고 하면 안 된다.

초기에는 오히려 작아야 한다.  
그리고 진심이 있어야 한다.

### 작은 진심의 정의

“작은 진심”은 거창한 인생 서사가 아니다.

- 7년째 쓰는 지갑을 아직 바꾸지 못하는 이유
- 비싸지 않은 시계를 계속 차는 이유
- 첫 월급으로 산 물건을 아직 보관하는 이유
- 매일 쓰는 펜이 손에 익은 이유
- 낡은 가방이 이상하게 나다운 이유

이 정도의 작은 이야기면 충분하다.

지님이 모아야 하는 것은 거창한 사연이 아니라,  
**사람이 실제로 지니고 살아온 작은 시간**이다.

---

## 7. 핵심 선언

MVP 인입 단계에서 지님은 이 선언을 분명히 해도 좋다.

> 지님은 모두를 위한 서비스가 아닙니다.

이 문장은 배타적으로 쓰면 위험하지만, 잘 쓰면 강력한 브랜드 장치가 된다.

중요한 건 “우리는 특별한 사람만 받는다”가 아니라,  
“우리는 물건을 자랑하는 방식과는 다른 태도를 지향한다”는 의미로 써야 한다.

### 추천 표현

> 지님은 모두를 위한 서비스가 아닙니다.  
> 오래 지닌 물건 하나에 담긴 이야기를 가진 사람을 위한 공간입니다.

또는 더 부드럽게:

> 모두에게 필요한 서비스는 아닐 수 있습니다.  
> 하지만 오래 지닌 물건 하나쯤 떠오르는 사람에게는, 꽤 의미 있는 공간이 되고 싶습니다.

MVP 랜딩/초대 페이지에서는 두 번째 문장이 더 자연스럽다.

## 9. MVP 인입 구조

초기 인입은 다음 흐름이 적합하다.

```text
1. 브랜드 메시지에 공감
2. 오래 지닌 물건 하나를 떠올림
3. 나의 첫 지님 만들기(CTA)
4. 사진 업로드
5. 물건 정보 입력
6. 질문 3개 답변
7. 지님 카드 생성
8. 취향 읽기
9. 공유 URL 복사
10. 소품함에 담기 등 → 카카오 로그인 유도(가입)
11. (회원) 추가 지님 기록, 소품함 취향 리포트 등
```

핵심은 2번이다.

사용자가 “내가 올릴 물건이 뭐가 있지?”를 떠올리는 순간, 지님은 이미 반쯤 성공한 것이다.

게스트는 **지님 카드 1장**까지 무료로 만들 수 있고, 수정·삭제·소품함 저장·추가 카드는 가입 후에 맞춘다(`mvp-spec-def.md` 1절).

## 10. 첫 지님 카드 등록 유도 전략

### 10.0 고객향 표현(취향·AI)

고객에게는 **“AI 취향 예측”** 같은 표현을 쓰지 않는다. 버튼·설명은 `mvp-spec-def.md` 3절의 고객향 용어(예: **취향 읽기**, “이 물건이 보여주는 취향”, “지님이 당신의 답변을 바탕으로 취향을 정리합니다”)를 따른다.

### 10.1 질문을 어렵게 만들지 않는다

초기 질문은 많으면 안 된다.
사용자가 “좋은 글을 써야 하나?”라고 느끼면 이탈한다.

### 추천 질문 3개

| 질문                      | 의도          |
| ----------------------- | ----------- |
| 이 물건은 무엇인가요?            | 진입 부담 낮추기   |
| 왜 아직도 지니고 있나요?          | 핵심 이야기 끌어내기 |
| 앞으로도 계속 지니고 싶은 이유가 있나요? | 지속성과 애착 확인  |

선택 질문으로 하나 추가 가능:

| 선택 질문                   | 의도          |
| ----------------------- | ----------- |
| 이 물건을 한 단어로 표현하면 무엇인가요? | 카드 태그/무드 생성 |

---

### 10.2 “좋은 글”이 아니라 “짧은 답”을 유도한다

사용자는 글을 잘 쓰고 싶어 하지 않는다.
다만 자기 이야기가 잘 정리되길 바란다.

그래서 입력 화면에서는 이렇게 말해야 한다.

```text
짧게 적어도 괜찮아요.
지님이 당신의 답변을 바탕으로 취향을 정리합니다.
```

단, AI가 사연을 지어낸다는 인상을 주면 안 된다.

```text
당신의 이야기를 대신 만들지 않습니다.
당신이 남긴 답을 보기 좋게 정리합니다.
```

---

### 10.3 결과물이 좋아야 한다

지님 카드가 예쁘고 저장하고 싶어야 한다.
초기 확산은 기능보다 결과물에서 나온다.

지님 카드는 다음 조건을 만족해야 한다.

| 조건           | 설명                         |
| ------------ | -------------------------- |
| 저장하고 싶다      | 회원·소품함 맥락에서 내 물건 기록으로 보관하고 싶은 카드(게스트는 소품함 저장 전 카카오 유도) |
| 공유하고 싶다      | SNS나 지인에게 보여줘도 과하지 않은 카드   |
| 자랑처럼 보이지 않는다 | 가격/브랜드보다 이유와 시간이 보임        |
| 내 이야기처럼 느껴진다 | 꾸며 낸 문장이 아니라 내 답변이 살아 있음 |
| 다시 만들고 싶다    | 가입 뒤 다른 물건도 지님으로 남기고 싶은 마음이 생김 |

## 11. 초기 사용자 모집 전략

### 11.1 1차 모집: 지인 기반 수동 모집

가장 먼저 할 일은 공개 홍보가 아니다.
네 주변에서 “작은 진심”을 가진 사람 10명을 찾는 것이다.

추천 대상:

* 시계 좋아하는 사람
* 문구 좋아하는 사람
* 사진/카메라 좋아하는 사람
* 오래 쓰는 물건이 있는 사람
* 자기 취향이 분명한 사람
* 인스타 자랑 문화에 피로감을 느끼는 사람

### 요청 메시지 예시

```text
내가 개인 프로젝트로 ‘지님’이라는 서비스를 만들고 있어.

비싼 물건 자랑이 아니라,
오래 지닌 물건에 담긴 이유를 기록하는 서비스야.

혹시 너에게 오래 지닌 물건 하나가 있다면
사진 한 장이랑, 왜 아직도 지니고 있는지 한두 줄만 보내줄 수 있을까?

내가 지님 카드 형태로 만들어서 보여줄게.
```

이 방식이 좋은 이유는 사용자가 직접 서비스를 쓰지 않아도,
“지님 카드 결과물”에 먼저 반응하게 만들 수 있기 때문이다.

### 11.3 3차 모집: 취향 커뮤니티/소규모 채널 공유

초기 카드는 공개적으로 많이 뿌리기보다, 관련 취향이 있는 작은 채널에 공유하는 것이 좋다.

추천 채널:

| 채널                 | 이유                    |
| ------------------ | --------------------- |
| 시계 커뮤니티/오픈채팅       | 초기 출발점과 잘 맞음          |
| 문구/만년필 커뮤니티        | 기록과 물건의 의미에 반응 가능     |
| EDC 커뮤니티           | “매일 지니는 물건” 콘셉트와 잘 맞음 |
| 빈티지/가죽소품 커뮤니티      | 오래 쓰는 물건의 감각과 맞음      |
| 개인 SNS/Threads     | 브랜드 철학 공유에 적합         |
| Product Hunt류는 후순위 | 너무 이른 단계              |

단, 이때도 “서비스 홍보”보다 “카드 모집”이 더 자연스럽다.

```text
작은 프로젝트로 ‘오래 지닌 물건의 이유’를 기록하는 지님 카드를 만들고 있습니다.
혹시 오래 지닌 물건 하나가 있다면 참여해주세요.
```

## 12. 초기 인입 퍼널

MVP 인입 퍼널은 아래처럼 설계한다.

```text
랜딩 방문
↓
브랜드 메시지 공감
↓
나의 첫 지님 만들기 CTA
↓
물건 사진 업로드
↓
물건 정보 입력
↓
질문 3개 답변
↓
지님 카드 생성
↓
취향 읽기
↓
공유 URL 복사
↓
소품함에 담기 등 → 카카오 로그인 유도
↓
(회원) 추가 지님·소품함 취향 리포트
```

선택적으로, 운영 초기에는 **다른 사람의 지님 카드 구경**(선별 아카이브 등)을 넣어 “이렇게 남기는구나”를 보여줄 수 있다.

### 퍼널별 체크 지표

| 단계         | 확인할 것           |
| ---------- | --------------- |
| 랜딩 방문      | 메시지 이해 여부       |
| CTA 클릭     | “나도 해볼까?” 반응 여부 |
| 사진 업로드     | 첫 행동 장벽         |
| 물건 정보 입력   | 이름·카테고리 입력 부담 |
| 질문 답변      | 답변 부담 수준        |
| 카드 생성      | 결과물 만족도         |
| 취향 읽기      | 1회성 결과에 대한 반응   |
| 공유 URL 복사   | 확산 가능성          |
| 소품함·가입 유도  | 카카오 전환, 가입 문구 수용도 |
| (회원) 추가 지님 | 가입 후 반복 기록 의지   |

## 13. 초기 제품 장치

지님 카드 등록을 유도하려면, 제품 안에 다음 장치가 필요하다.

### 13.1 예시 카드 먼저 보여주기

사용자는 처음에 무엇을 써야 할지 모른다.
그래서 예시가 필요하다.

예시 카드는 고가 물건보다 일상적인 물건이 좋다.

추천 예시:

| 물건          | 이유               |
| ----------- | ---------------- |
| 오래 찬 카시오 시계 | 비싸지 않아도 의미 있는 물건 |
| 손에 익은 지갑    | 매일 지니는 감각        |
| 오래 쓰는 펜     | 기록과 연결           |
| 낡은 안경       | 일상과 정체성          |
| 첫 월급으로 산 물건 | 기억과 시간           |

---

### 13.2 “비싸지 않아도 좋습니다” 메시지

이 문장은 반드시 필요하다.

지님이 사치품 자랑 서비스가 아니라는 점을 가장 빨리 전달한다.

추천 위치:

* 첫 지님 등록 화면
* 예시 카드 섹션
* 캠페인 안내
* 공유 카드 하단

추천 문구:

```text
비싸지 않아도 좋습니다.
오래 지닌 이유가 있다면 충분합니다.
```

---

### 13.3 공개 범위 선택

인스타 자랑을 싫어하는 사람들은 공개에 민감할 수 있다.

따라서 처음부터 공개/비공개 선택이 있어야 한다.

```text
나만 보기
링크로만 공유
공개하기
```

초기 MVP에서는 최소한 아래 두 개만 있어도 된다.

```text
나만 보기
링크로 공유
```

이게 있으면 사용자는 부담 없이 기록할 수 있다.

---

### 13.4 결과 카드 다운로드

초기 확산을 위해서는 카드 이미지 다운로드가 중요하다.

기능:

* 카드 PNG 저장
* 링크 복사
* SNS 공유용 이미지 저장

다만 공유 문구는 과시형이 아니어야 한다.

추천 공유 문구:

```text
내가 오래 지닌 물건 하나를 기록했습니다.
```

```text
나의 첫 지님을 남겼습니다.
```

```text
비싸지 않아도, 오래 지닌 이유가 있는 물건.
```

## 14. 커뮤니티를 바로 열지 않는 전략

MVP 단계에서 커뮤니티를 바로 열면 위험하다.

커뮤니티가 열리면 사용자는 자연스럽게 비교한다.

* 누가 더 좋은 물건을 가졌나
* 누가 더 비싼 물건을 올렸나
* 누가 더 많은 반응을 받았나

이는 지님의 철학과 충돌한다.

따라서 MVP 단계에서는 커뮤니티보다 **선별된 지님 카드 아카이브**가 적합하다.

### 추천 방식

| 방식        | 설명                          |
| --------- | --------------------------- |
| 공개 피드 없음  | 초기에는 전체 공개 피드 운영하지 않음       |
| 선별 카드 소개  | 운영자가 좋은 지님 카드를 큐레이션         |
| 주제별 모음    | “오래 쓴 물건”, “비싸지 않아도 좋은 것” 등 |
| 좋아요 없음    | 반응 경쟁 최소화                   |
| 조회수/랭킹 없음 | 과시 구조 방지                    |

초기에는 “커뮤니티”보다 “작은 전시”에 가까워야 한다.

---

## 15. 첫 공개 카드 수집 전략

MVP 출시 전후로 운영자가 직접 좋은 카드를 모아야 한다.

### 목표 수량

| 단계      |   카드 수 |
| ------- | -----: |
| 내부 샘플   |     5개 |
| 지인 테스트  | 10~20개 |
| 소프트 런칭  | 30~50개 |
| 공개 아카이브 | 50개 이상 |

### 카드 구성 비율

초기 공개 카드가 고가 시계 중심이면 브랜드가 왜곡된다.
따라서 의도적으로 다양성을 구성해야 한다.

| 카테고리     |  비율 |
| -------- | --: |
| 시계       | 25% |
| 지갑/가죽소품  | 20% |
| 펜/문구     | 20% |
| 안경/가방/키링 | 20% |
| 기타 개인 물건 | 15% |

### 가격 다양성 원칙

초기 공개 카드에는 반드시 저가/일상 물건이 포함되어야 한다.

* 만 원대 시계
* 오래 쓴 볼펜
* 낡은 카드지갑
* 기념품 키링
* 고장 났지만 못 버리는 물건

이런 카드가 있어야 지님이 “가격이 아니라 이유”를 본다는 점이 전달된다.

지님은 모두를 위한 서비스는 아닐 수 있습니다.

당신에게도 오래 지닌 물건 하나가 있다면,

```

## 17. 초기 운영 원칙

MVP 인입 이후 운영은 다음 원칙을 따른다.

| 원칙                | 설명                    |
| ----------------- | --------------------- |
| 첫 사용자는 많이보다 깊게    | 가입자 수보다 좋은 카드와 반응을 우선 |
| 가격보다 이유를 보여주기     | 고가 물건을 전면에 세우지 않음     |
| 카드 품질을 운영자가 관리    | 초반에는 선별/큐레이션 필요       |
| 공유는 하되 경쟁은 만들지 않기 | 좋아요/랭킹/조회수 노출 최소화     |
| 사용자의 원문을 존중       | 답변을 과하게 바꾸지 않음(`mvp-spec-def.md` 3절 톤과 맞출 것) |
| 공개 부담을 낮추기        | 나만 보기/링크 공유 옵션 제공     |
| 로그인은 카카오 우선      | “카카오로 소품함 만들기” 등 `mvp-spec-def.md` 2절 문구 사용 |
| 커뮤니티는 천천히         | 초기에는 아카이브와 카드 생성 중심   |

## 18. 핵심 검증 질문

MVP 출시 후에는 다음 질문에 답해야 한다.

### 사용자 반응

* 사람들이 오래 지닌 물건을 쉽게 떠올리는가?
* 사진을 올리는 데 부담이 없는가?
* 물건 이름·카테고리 입력이 부담되지 않는가?
* 질문 3개에 답하기 쉬운가?
* 완성된 카드가 마음에 드는가?
* **취향 읽기** 결과가 부담 없이 와닿는가?
* 공유 URL을 복사하거나 퍼뜨리고 싶은가?
* 소품함·추가 지님을 위해 **카카오 로그인**으로 이어지는가?
* (가입 후) 다른 물건도 기록하고 싶은가?

### 브랜드 반응

* 지님이 자랑 서비스가 아니라는 점이 전달되는가?
* 비싸지 않은 물건도 올려도 된다고 느끼는가?
* 사용자가 자기 이야기를 꺼내고 싶어 하는가?
* 작은 진심을 가진 사람들이 모이는가?

## 20. 최종 전략 문장

지님의 MVP 인입 전략은 많은 사람을 모으는 것이 아니다.

**오래 지닌 물건 하나에 담긴 작은 진심을 가진 사람들을 모으는 것**이다.

따라서 지님은 처음부터 이렇게 말해야 한다.

> 지님은 모두를 위한 서비스가 아닙니다.
> 하지만 오래 지닌 물건 하나쯤 떠오르는 사람에게는, 자기 시간을 기록하는 작은 공간이 될 수 있습니다.

MVP의 목표는 거대한 커뮤니티가 아니다.

> 한 사람이 자신의 물건 하나를 다시 바라보고,
> 그 이유를 남기고,
> 완성된 지님 카드를 보고
> “이건 내 이야기다”라고 느끼게 만드는 것.

그 경험이 만들어진다면,
지님은 충분히 세상에 나올 의미가 있다.

````
