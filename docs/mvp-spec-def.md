# 지님 MVP 정책 확정 사항

## 1. 게스트 카드 정책

게스트는 가입 없이 **지님 카드 1장**을 만들 수 있다.

### 가능
- 물건 사진 업로드
- 물건 이름/카테고리 입력
- 질문 3개 답변
- 지님 카드 생성
- 카드 공유 URL 복사
- 1회성 취향 읽기 결과 확인

### 불가
- 카드 수정
- 카드 삭제
- 소품함 저장
- 추가 카드 생성
- 내 소품함 URL 생성

### 가입 유도 시점
게스트가 아래 행동을 하려 할 때 가입을 유도한다.

- 카드를 수정하려 할 때
- 카드를 소품함에 담으려 할 때
- 지님을 하나 더 만들려 할 때
- 취향 리포트를 더 자세히 보고 싶어 할 때

가입 유도 문구는 “가입하세요”보다 아래처럼 간다.

> 이 카드를 내 소품함에 보관하시겠어요?  
> 카카오로 간편하게 소품함을 만들 수 있습니다.

---

## 2. 로그인 정책

로그인은 **카카오 간편 로그인**을 우선 적용한다.

### 이유
- 국내 사용자에게 가장 익숙함
- 지님이 한국어 기반 서비스로 시작하므로 자연스러움
- 이메일/비밀번호보다 진입 부담이 낮음
- “소품함 만들기” 흐름과 잘 맞음

### 로그인 진입 문구

> 카카오로 소품함 만들기

또는

> 카카오로 시작하기

---

## 3. 취향 분석 기능 명칭

고객향 표현에서는 **“AI 취향 예측”**이라는 표현을 쓰지 않는다.

이 표현은 기능적으로는 맞지만, 지님 톤에는 다소 차갑고 분석 서비스처럼 느껴진다.

### 내부 용어
- AI 취향 예측
- Taste inference
- Taste report generation

### 고객향 용어
아래 방향으로 사용한다.

| 상황 | 추천 표현 |
|---|---|
| 게스트 카드 1장 기반 | 이 물건이 보여주는 취향 |
| 게스트 결과 섹션 | 나의 지님 취향 |
| 회원 소품함 기반 | 소품함 취향 리포트 |
| 버튼/CTA | 취향 읽기 |
| 설명 문구 | 지님이 당신의 답변을 바탕으로 취향을 정리합니다 |

---

## 4. 게스트 취향 읽기

게스트는 카드 1장을 만든 뒤, 1회성으로 가벼운 취향 읽기 결과를 볼 수 있다.

### 결과 구성

| 항목 | 설명 |
|---|---|
| 취향 키워드 | 카드 답변에서 드러난 2~3개 키워드 |
| 물건을 고르는 기준 | 이 사람이 어떤 기준으로 물건을 고르는지 |
| 다음에 기록해볼 물건 | 비슷한 결의 다음 지님 제안 |

### 예시 문구

> 이 물건에서 당신은 단정함과 오래감을 중요하게 보는 취향이 드러납니다.  
> 화려한 새로움보다, 매일 곁에 두어도 질리지 않는 물건을 선호하는 편입니다.

---

## 5. 회원 소품함 취향 리포트

회원은 여러 개의 지님을 등록한 뒤, 더 정교한 취향 리포트를 볼 수 있다.

### MVP 기준
- 등록된 지님 2개 이상부터 리포트 제공
- 최대 5개 소품 기준으로 분석
- 리포트는 너무 거창하지 않게 제공

### 결과 구성

| 항목 | 설명 |
|---|---|
| 나의 취향 키워드 | 반복적으로 나타나는 태그/의미 |
| 내가 물건을 고르는 기준 | 답변에서 드러나는 선택 기준 |
| 내 소품함의 분위기 | 단정함, 실용성, 빈티지, 기념성 등 |
| 다음에 기록해볼 지님 | 아직 비어 있는 카테고리나 어울리는 물건 제안 |
